## What is GitTrace?

GitTrace sits inside your browser and analyzes any GitHub repository you visit. It injects a live score badge into the GitHub header and gives you a full breakdown of:

- **AI Probability** — how likely the code was generated by an LLM
- **License Risk** — whether the license is safe, needs review, or is legally dangerous
- **Security Issues** — known CVEs, phantom packages, and hardcoded secrets
- **System Compatibility** — whether the repo will run on your OS and runtime

## Features Built

### AI Detection
- Per-file AI probability scoring via Sapling API
- Text chunking for large files (1500 char chunks)
- Heuristic boosters: generic variable density, comment ratio, boilerplate patterns, human signals
- Weighted overall score (larger files get more weight)
- Score label: Low / Medium / High / Very High

### Commit Velocity Analysis
- Large first commit detection (> 1000 lines)
- Bulk commit flagging (> 500 lines per commit)
- Bulk delete-and-rewrite detection (vibe coding pattern)
- Generic commit message detection
- Single-author bulk commit pattern

### License Compliance
- 25 SPDX license identifiers classified
- Risk levels: SAFE / REVIEW / HIGH_RISK / UNKNOWN
- Combined risk warning: high AI score + GPL/AGPL = critical alert
- Covers: MIT, Apache, GPL-2/3, AGPL, LGPL, MPL, EPL and more

### Security Shield
- **CVE Scanning** — Google OSV API across npm, PyPI, Go, Rust, RubyGems
- **Phantom Package Detection** — checks npm and PyPI registries for hallucinated names
- **Secret Scanner** — 25 patterns: AWS keys, GitHub tokens, Stripe keys, DB URLs, private keys, JWT secrets
- False positive filtering for placeholders and test values
- Masked secret display (never shows actual value)

### File Tree Heatmap
- Colour-coded dots injected next to every file in GitHub's file browser
- Green (< 30%) → Amber (30–60%) → Orange (60–80%) → Red (> 80%)
- Hover tooltip shows exact score and file path
- MutationObserver handles dynamic GitHub rendering
- Cleaned up on SPA navigation

### Chrome Extension UI
- Shadow DOM badge injected into GitHub repo header
- Animated score ring (SVG, fills on result)
- 4-tab dropdown: Score, Breakdown, Security, Compat
- Loading skeleton shimmer state
- Error state with friendly messages
- Refresh button to force re-scan
- SPA navigation support (turbo:load, pjax:end, popstate)
- Session cache (10 min TTL) to avoid redundant API calls

## Getting Started

### Prerequisites

```bash
node --version   # v18 or higher required
git --version
```

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/gittrace.git
cd gittrace
```

### 2. Set up the backend

```bash
cd backend
npm install
cp .env.example .env
```

Edit `backend/.env` and fill in your keys:

```bash
GITHUB_TOKEN=your_github_pat_here        # github.com/settings/tokens
AI_API_KEY=your_sapling_key_here         # sapling.ai/user/settings
GITTRACE_SECRET=dev-secret-day3
```

Start the server:

```bash
npm run dev
# Server running at http://localhost:3001
```
